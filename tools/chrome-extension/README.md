# nyan Real Audio Wall connector (Chrome拡張)

タブの音声とウィンドウ位置を OBS の nyan Real Display Wall（空間音響
(Audio Wall) グループ）へローカル WebSocket（既定 `ws://127.0.0.1:8796`）で
送る拡張です。仮想オーディオケーブル等のドライバーは不要です。

## 仕組み

- content script がページの `<audio>/<video>` を `createMediaElementSource()`
  でフックします。この時点で**タブの音は OS の出力デバイスから切り離され**、
  拡張経由 → OBS のモニタリングだけが音の出口になります（二重再生なし）
- PCM のキャプチャは AudioWorklet（音声スレッド、約 21ms バッチ）で行い、
  ページのメインスレッドが重くても途切れません。ページの CSP で Worklet
  モジュールが読めない場合は ScriptProcessor（約 43ms）へ自動フォール
  バックします
- PCM は 16bit でサービスワーカーへ送り、1 本の WebSocket に
  多重化して Audio Wall へ届きます
- サービスワーカーは `chrome.windows` / `chrome.system.display` で
  ウィンドウ位置を監視し、仮想デスクトップ上の正規化 X 位置（-0.5〜+0.5）を
  メタデータとして送ります。Audio Wall 側が仮想スクリーンの実際の幾何
  （距離・物理幅・湾曲）から方位角を厳密に計算し、頭の向きに追従する
  空間音響として鳴らします
- `windows.onBoundsChanged` は移動の確定時にしか発火しないため、
  **ドラッグ中の追従**はサービスワーカーが `chrome.windows.get` を
  200ms でポーリングして補完します。窓を掴んで動かすと音の方向が
  リアルタイムに付いてきます
- ウィンドウの移動・タブの別ウィンドウへの移動にも追従します

## i18n

表示文言は `_locales/`（既定 `en`、`ja` 同梱）の `chrome.i18n` で解決され、
ブラウザの UI 言語に従います。manifest の名前・説明も `__MSG_*__` 参照の
ため、ストア掲載文も言語別に出ます。文言を増やすときは `en` と `ja` の
両方の `messages.json` に同じキーを足してください（拡張名はブランドとして
全言語共通）。

## インストール

1. Chrome / Edge で `chrome://extensions` を開く
2. 「デベロッパーモード」を ON
3. 「パッケージ化されていない拡張機能を読み込む」でこのフォルダーを選択

## 使い方

1. OBS 側: nyan Real Display Wall ソースのプロパティで「空間音響
   (Audio Wall)」を有効にする（ブラウザ連携ポートは既定 8796。変える場合は
   拡張のオプションも同じ番号に）
2. ブラウザで音を再生する — それだけです。同プロパティの
   「現在キャプチャ中」に `[WS] <タブ名>` が出ます

## OBS がいないとき

OBS（の WebSocket 受け口）に接続できない間、タブの音は**通常どおり
ローカルで再生**されます。OBS が起動すると 3 秒以内に自動で OBS 経由の
空間音響に切り替わり、OBS を終了するとローカル再生に戻ります。

## 制約

- DRM 保護メディア（Netflix 等）は Web Audio を通れないため無音になります
- ページが自前の Web Audio で既に要素を取り込んでいる場合はフックしません
- Web Audio を直接使うサイト（ブラウザゲーム等）の音はタブ要素フックでは
  取れません
- 同一プロファイル内のタブ単位で分離できます（ウィンドウ単位の位置 ×
  タブ単位の音声）
